# Web Agency System — Master Instructions

You are building a premium static HTML website for a client. This system produces websites that look hand-crafted by a design agency, not generated by AI. Every decision you make should pass this test: "Would a senior designer at a top agency approve this?"

Read CLIENT_BRIEF.md, MEDIA.md, and the selected DESIGN.md before writing any code.

---

## Workflow

Follow these phases in order. Do not skip phases.

### Phase 1: Competitor Research

Use Firecrawl to scrape all competitor URLs listed in CLIENT_BRIEF.md. Analyze:
- What sections/pages do competitors have?
- What features are standard in this niche? (booking, pricing tables, galleries, etc.)
- What social proof do they use? (reviews, logos, certifications)
- What are they doing poorly? (slow load, bad mobile, generic copy, weak CTAs)
- What keywords/phrases does this niche use?

Write a brief competitive analysis (10-15 lines) as a comment block at the top of your first working file, then delete it before final delivery. Use these insights to ensure the client's site matches or exceeds every competitor.

### Phase 2: Design System Assembly

1. Check `design-systems/design-md/` for a DESIGN.md that matches the client's industry or aesthetic (71+ brands available — browse subfolder names to find a match)
2. If client media exists (MEDIA.md), examine the imagery descriptions and pull color cues from them — warm photos mean warm palette, cool/clinical photos mean cool palette
3. Use the Google Fonts skill to select a distinctive typography pairing. NEVER use: Inter, Roboto, Open Sans, Lato, Montserrat, Poppins, or Nunito. These are AI-default fonts. Choose fonts with character — serif + geometric sans, display + monospace, or variable weight fonts
4. Assemble the project's DESIGN.md with: color palette (6-8 colors as CSS custom properties), typography (with clamp() fluid scale), spacing scale, border-radius values, and animation timing preferences
5. The design system must specify exact CSS values, not descriptions. "border-radius: 0" is a constraint. "Modern and clean" is not.

### Phase 3: Build

Build a static HTML/CSS/JS website. Output structure:
```
project-name/
├── index.html
├── [additional pages].html
├── css/
│   └── styles.css
├── js/
│   └── main.js
├── images/
│   └── [optimized assets]
└── fonts/ (if self-hosting)
```

**Use CDN links for all libraries** (GSAP, ScrollTrigger, Lenis, Barba.js, Vanta.js). Do not use npm/build tools. These are static sites deployed to shared hosting.

### Phase 4: Review in Browser

Preview the site using Claude in Chrome. Check:
- Does it look premium, not generic?
- Do all animations work smoothly?
- Is it responsive at 360px, 768px, 1024px?
- Are there any layout shifts or janky transitions?
- Does the copy follow PAS framework?
- Is the primary CTA above the fold?

Fix any issues before declaring the build complete.

### Phase 5: Deploy

Use the deploy script in `scripts/` to push to Hostinger via SFTP, or guide the user through manual upload.

## Design Rules (non-negotiable)

### Typography
- Always use `clamp()` for fluid sizing. Never use fixed px with media query jumps.
- Heading scale should have 3x+ size jumps between body and h1, not 1.5x.
- Line-height: 1.1-1.2 for headings, 1.6-1.8 for body text.
- Letter-spacing: slight negative (-0.02em) on large headings for tightness.
- Font weights: use extremes (300 + 800) not middle values (400 + 600).
- All fonts must be loaded via Google Fonts link tags with `display=swap`.

Example fluid scale:
```css
:root {
  --text-sm: clamp(0.875rem, 0.8rem + 0.25vw, 1rem);
  --text-base: clamp(1rem, 0.9rem + 0.35vw, 1.125rem);
  --text-lg: clamp(1.25rem, 1rem + 0.75vw, 1.5rem);
  --text-xl: clamp(1.5rem, 1rem + 1.5vw, 2.25rem);
  --text-2xl: clamp(2rem, 1.2rem + 2.5vw, 3.5rem);
  --text-3xl: clamp(2.5rem, 1rem + 4vw, 5rem);
}
```

### Layout
- NEVER use three equal cards in a row. Use asymmetric bento grids — one card spanning 2 columns, varied heights, accent cards.
- Use CSS Grid with `grid-template-columns: repeat(auto-fit, minmax(300px, 1fr))` as a baseline, then customize per section.
- Generous whitespace. Sections should have `padding-block: clamp(4rem, 8vw, 8rem)`.
- Max content width: 1200px-1400px, centered.
- Break out of the container occasionally — full-bleed images, offset elements, overlapping sections. Monotonous containment is a template giveaway.

### Color
- Define all colors as CSS custom properties on `:root`.
- Every palette needs: primary, primary-dark, accent, background, surface, text-primary, text-secondary, text-on-primary.
- Use `backdrop-filter: blur()` for glass/depth effects sparingly — on nav overlays, floating cards, modal backgrounds. Not on everything.
- Dark sections should use actual rich dark colors (#0a0a0a, #111827) not pure black (#000).

### Images & Media
- **Client media always takes priority** over stock photos. Check MEDIA.md first.
- Only use Unsplash/Pexels for gaps where no client media exists. Search for specific, relevant terms, not generic ones.
- All images must use: `loading="lazy"`, `width`/`height` attributes, `alt` text.
- Use `<picture>` with WebP source + fallback where possible.
- Hero images/videos should be full-viewport or near-full.
- If client provides video tagged for hero use in MEDIA.md, implement as background video with `autoplay muted loop playsinline` and a poster fallback image.

### SVG & Graphics
- Generate custom SVG graphics for decorative elements — geometric patterns, wave dividers, abstract backgrounds. Never use generic clip-art.
- Icons should be inline SVG for styling control, not icon font libraries.
- Use consistent stroke width and style across all icons (outline OR filled, not mixed).

## Motion Rules (non-negotiable)

### Smooth Scroll
Always initialize Lenis for smooth scrolling:
```html
<script src="https://cdn.jsdelivr.net/npm/lenis@latest/dist/lenis.min.js"></script>
```
Sync with GSAP ticker for ScrollTrigger compatibility.

### Scroll Animations (GSAP + ScrollTrigger)
Every section gets an entrance animation. Use `stagger` for groups of elements.
```html
<script src="https://cdn.jsdelivr.net/npm/gsap@3/dist/gsap.min.js"></script>
<script src="https://cdn.jsdelivr.net/npm/gsap@3/dist/ScrollTrigger.min.js"></script>
```
- Sections: fade up (opacity 0→1, y 40→0) over 0.8s with ease "power2.out"
- Cards/grid items: stagger 0.1-0.15s
- Hero heading: split into words/characters, stagger reveal 0.03s per character
- Images: subtle scale (1.05→1) or clip-path reveal
- Counters/stats: animate numbers counting up when scrolled into view
- Start trigger: `top 85%` (fires just before element enters viewport)
- Do NOT animate everything. Decorative backgrounds, nav, and footer are static.

### Page Transitions (Barba.js)
Implement smooth transitions between pages:
```html
<script src="https://cdn.jsdelivr.net/npm/@barba/core@latest/dist/barba.umd.js"></script>
```
- Wrap page content in `[data-barba="wrapper"]` > `[data-barba="container"]`
- Use a fade or slide transition (0.4-0.6s) between pages
- Re-initialize scroll position and animations after transition

### Hero Background (Vanta.js — when appropriate)
For businesses where a dynamic background fits (tech, wellness, premium services):
```html
<script src="https://cdn.jsdelivr.net/npm/three@0.134.0/build/three.min.js"></script>
<script src="https://cdn.jsdelivr.net/npm/vanta@latest/dist/vanta.fog.min.js"></script>
```
- Match the Vanta effect to the business: fog/waves for wellness/spa, net/topology for tech, birds for nature/outdoor
- Use brand colors for the effect, not defaults
- Keep opacity subtle — the background should enhance, not distract
- Always provide a CSS gradient fallback for non-WebGL browsers
- If Vanta is not appropriate for the niche (restaurant, trades, retail), skip it entirely and use a strong hero image or video instead

### Micro-interactions
- CTA buttons: magnetic hover (cursor pull effect), scale 1.02-1.05 on hover, smooth color transition
- Cards: subtle translateY(-4px) and shadow increase on hover
- Nav links: underline slide-in from left on hover, not instant underline
- Form inputs: label float animation on focus, border color transition
- Scroll-to-top button: fade in after scrolling past first section
- Mobile menu: slide-in from right with staggered link animation

### Page Preloader
Include a brief loading animation:
- Full-screen overlay with brand color or dark background
- Simple animation: logo fade-in, progress bar, or geometric shape
- Duration: 1-2 seconds max, then slide up to reveal page
- Triggers hero entrance animation after preloader completes
- Use `window.addEventListener('load', ...)` to dismiss

### Timing & Easing
- Default ease: "power2.out" for entrances, "power2.inOut" for transitions
- Duration: 0.6-1.0s for section entrances, 0.3-0.4s for hovers, 0.4-0.6s for page transitions
- Stagger: 0.08-0.15s between items
- NEVER use linear easing for UI animations. NEVER exceed 1.5s for any single animation.

## Conversion Rules (non-negotiable)

### Copywriting — PAS Framework
Every page hero section must follow Problem → Agitate → Solution:
- **Problem**: Name the specific pain the target audience experiences (from CLIENT_BRIEF.md)
- **Agitate**: Make them feel the cost of not solving it (1-2 sentences)
- **Solution**: Position the client's service as the answer, with a specific benefit and clear CTA

NEVER write vague aspirational copy like:
- "Transform your experience"
- "Unlock the future"
- "Your journey starts here"
- "Welcome to [Business Name]"
- "We are committed to excellence"

ALWAYS write specific, benefit-driven copy:
- "Your back pain shouldn't dictate your weekday. 45 minutes on the table and you'll feel the difference before you reach your car."
- "3 out of 4 Manchester restaurants fail health inspections on ventilation. We've helped 200+ pass first time."

### CTAs
- Use first-person language: "Book my session" not "Book a session", "Get my free quote" not "Get a quote"
- One primary CTA per section — never compete with yourself
- Primary CTAs: centered, high-contrast button with ample padding (16px 32px minimum)
- Above-fold: headline + value proposition + CTA + social proof must ALL be visible without scrolling
- Secondary CTAs: text links or ghost buttons, visually subordinate

### Social Proof
- Place social proof immediately below the hero section
- Client logos (if applicable) in a subtle grayscale row
- Testimonials: 3-5 on homepage, with real names and context. Pull from competitor research if client hasn't provided their own yet (mark as placeholder)
- Star ratings / review counts if available
- "As seen in" / certifications / awards
- Specific numbers: "200+ clients served", "4.9 rating from 127 reviews", "15 years experience"

### Sticky Mobile CTA
On mobile (< 768px), implement a sticky bottom bar:
- Fixed to bottom of viewport, 60px height
- Brand-colored background, white text
- Phone number (click-to-call via `tel:` link) OR primary booking CTA
- Appears after scrolling past the hero section
- Subtle shadow to separate from content
- z-index above all content

### Lead Capture
- Contact forms: maximum 4-5 fields (name, email, phone, message, optional dropdown)
- Submit button uses benefit language: "Get my free quote" not "Submit"
- Include a privacy reassurance line below the form: "We'll get back to you within 24 hours. No spam, ever."

## Technical Standards (non-negotiable)

### HTML
- Semantic HTML5: `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<footer>`
- Proper heading hierarchy: one `<h1>` per page, then `<h2>` → `<h3>` in order
- All interactive elements must be keyboard-accessible
- Skip-to-content link for accessibility
- Language attribute on `<html>`

### SEO & Structured Data
Every page must include:
```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "LocalBusiness",
  "name": "...",
  "address": { "@type": "PostalAddress", ... },
  "telephone": "...",
  "openingHoursSpecification": [...],
  "image": "...",
  "url": "...",
  "sameAs": [...]
}
</script>
```
Plus FAQ schema on pages with FAQ sections. Plus Review/AggregateRating if reviews are displayed.

Every page must have:
- Unique `<title>` tag (format: "Page Name | Business Name — Location")
- `<meta name="description">` (150-160 chars, includes location + primary service)
- Open Graph tags (og:title, og:description, og:image, og:url, og:type)
- Twitter Card tags
- Canonical URL
- Favicon (generate as inline SVG in a `<link>` tag)

### Performance
- Total page weight under 2MB (excluding hero video if client-provided)
- All CSS in one file, all JS in one file (or two: main.js + libraries via CDN)
- Images: WebP format, lazy-loaded, with explicit width/height to prevent layout shift
- Fonts: preconnect to Google Fonts, use `display=swap`
- Critical CSS for above-fold content should be in a `<style>` tag in `<head>`
- Defer non-critical JS: `<script defer>`

### Responsive
- Mobile-first CSS (base styles = mobile, then `@media (min-width: ...)` for larger)
- Test at: 360px (small phone), 768px (tablet), 1024px (laptop), 1440px (desktop)
- Tap targets: minimum 44x44px on mobile
- No horizontal scroll at any viewport
- Navigation: hamburger menu on mobile with slide-in overlay

### 404 Page
Create a simple, branded 404 page with:
- The site's header/nav
- A clear "Page not found" message
- A link back to the homepage
- Consistent with the site's design system

## File Naming & Organization
- HTML files: lowercase, hyphens (about-us.html, not About Us.html)
- CSS: single `styles.css` file with logical sections commented
- JS: single `main.js` file with initialization at bottom
- Images: descriptive names (sauna-interior-wide.webp, not IMG_4392.webp)
- All paths relative (no absolute URLs except for CDN links)

---

## What NOT to do
- No CSS frameworks (Bootstrap, Tailwind via CDN). Write custom CSS. Frameworks are a template giveaway.
- No icon font libraries (Font Awesome). Use inline SVG.
- No Lorem Ipsum — ever. Write real copy using the PAS framework and client brief.
- No placeholder images labeled "placeholder" — use Unsplash/Pexels or client media.
- No hamburger menu on desktop. Only on mobile/tablet.
- No "Copyright 2024" — use the current year dynamically with JS.
- No external widget embeds (Trustpilot widgets, Google review widgets) unless specifically requested. Build the social proof into the design natively.
- No cookie banners unless specifically requested (static HTML sites with no tracking don't need them).
